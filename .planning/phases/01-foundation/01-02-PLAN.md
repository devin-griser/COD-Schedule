<task type="auto">
  <name>Task 1: Create PandaScore fetch script and validate API live</name>
  <files>scripts/fetch-schedule.ts</files>
  <action>
    Create `scripts/fetch-schedule.ts` — a reusable Node.js script that:

    1. Reads `PANDASCORE_TOKEN` from environment (via `process.env`). If the token is missing, print a clear error: "Set PANDASCORE_TOKEN in .env.local" and exit with code 1.

    2. Calls both PandaScore CODMW endpoints:
       - `GET https://api.pandascore.co/codmw/matches/upcoming?per_page=50`
       - `GET https://api.pandascore.co/codmw/matches/past?per_page=50`
       Both with `Authorization: Bearer ${token}` header.

    3. Logs diagnostic info to console:
       - Total matches returned from each endpoint
       - Unique league names found (to identify CDL vs other CoD leagues)
       - Number of matches after filtering for CDL league
       - Sample of first match object's league/serie/tournament fields (for inspection)

    4. Filters matches for CDL only: `m.league?.name` should contain "Call of Duty League" (verify exact string with first response). Log if zero CDL matches found after filtering.

    5. Normalizes each PandaScore match to the `Match` interface:
       - `id`: String(m.id)
       - `status`: Map PandaScore statuses — `not_started` → `scheduled`, `running` → `live`, `finished` → `completed`. Unknown → `scheduled`.
       - `scheduledAt`: `m.scheduled_at || m.begin_at`
       - `homeTeam`/`awayTeam`: Normalize from `m.opponents` array. If opponent is missing, use `'TBD'`. Create TeamRef with: `id` = kebab-case slug of name, `name` = opponent.name, `shortName` = derive from name (last word or known abbreviation).
       - `scores`: If status is completed and results exist, `{ home: m.results[0]?.score ?? 0, away: m.results[1]?.score ?? 0 }`. Otherwise `undefined`.
       - `streamUrl`: `m.streams_list?.[0]?.raw_url ?? 'https://www.twitch.tv/callofdutyleague'`
       - `tournament`: `{ name: m.league?.name ?? 'CDL 2026', stage: m.serie?.full_name ?? '', round: m.tournament?.name ?? '' }`

    6. Writes the normalized data to `src/data/schedule.json` in the ScheduleData format:
       ```json
       { "version": "1", "generatedAt": "...", "matches": [...] }
       ```

    7. Prints a summary: "Wrote N matches to src/data/schedule.json"

    The script must be runnable with: `npx tsx scripts/fetch-schedule.ts`

    Load `.env.local` manually at the top of the script using:
    ```typescript
    import { readFileSync } from 'fs'
    import { resolve } from 'path'

    // Load .env.local
    try {
      const envPath = resolve(process.cwd(), '.env.local')
      const envContent = readFileSync(envPath, 'utf-8')
      for (const line of envContent.split('\n')) {
        const [key, ...valueParts] = line.split('=')
        if (key && valueParts.length > 0) {
          process.env[key.trim()] = valueParts.join('=').trim()
        }
      }
    } catch { /* .env.local not found, rely on existing env */ }
    ```

    This avoids adding a `dotenv` dependency.

    After creating the script, run it: `npx tsx scripts/fetch-schedule.ts`

    Capture and analyze the output carefully. The diagnostic logging is the primary deliverable — it answers whether PandaScore has 2026 CDL data.
